# IP验证登录系统 - 设计文档

## 1. 技术架构
- 语言：Python 3.11+
- 数据库：SQLite
- HTTP 服务：Python 标准库 `http.server`
- 数据访问：`sqlite3`
- 密码哈希：`hashlib.pbkdf2_hmac`
- TOTP：RFC 6238（标准库实现）

## 2. 总体架构
- 接入层：Nginx（`auth_request`）
- 认证服务层：IPAuth Service（策略引擎、身份认证、会话管理、地点管理）
- 存储层：SQLite（用户、地点、IP 绑定、会话、审计日志）

## 3. 模块划分
### 3.1 `ipauth/db.py`
- 数据库初始化与连接管理
- 表结构创建

### 3.2 `ipauth/security.py`
- 密码哈希与校验
- TOTP 生成与校验
- 会话 Token 生成与摘要

### 3.3 `ipauth/policy.py`
- 规则引擎
- 输出决策：`ALLOW` / `CHALLENGE_ONE_OF` / `CHALLENGE_BOTH`
- 输出标志：`require_location_rebind`

### 3.4 `ipauth/server.py`
- HTTP 路由与请求处理
- `/auth/check`：给 Nginx 调用
- `/auth/login`：挑战验证并签发会话
- `/auth/locations`、`/auth/bindings`：管理接口

## 4. 规则引擎设计
输入维度：
- 是否同 IP
- Cookie 状态（`valid` / `expired` / `missing` / `invalid`）
- 当前地点是否公共场所

输出规则：
1. 同 IP + Cookie valid => `ALLOW`
2. 同 IP + Cookie expired => 非公共场所 `ALLOW`；公共场所 `CHALLENGE_ONE_OF`
3. 同 IP + Cookie missing/invalid => `CHALLENGE_ONE_OF`
4. 不同 IP + Cookie valid => `CHALLENGE_ONE_OF` + `require_location_rebind=true`
5. 不同 IP + Cookie expired/missing/invalid => `CHALLENGE_BOTH`

## 5. SQLite 数据模型
### 5.1 `users`
- `id` INTEGER PK
- `username` TEXT UNIQUE
- `password_hash` TEXT
- `totp_secret` TEXT
- `created_at` INTEGER

### 5.2 `locations`
- `id` INTEGER PK
- `user_id` INTEGER
- `name` TEXT
- `is_public` INTEGER
- `created_at` INTEGER

### 5.3 `ip_location_bindings`
- `id` INTEGER PK
- `user_id` INTEGER
- `ip` TEXT
- `location_id` INTEGER
- `last_verified_at` INTEGER
- `UNIQUE(user_id, ip)`

### 5.4 `sessions`
- `id` INTEGER PK
- `user_id` INTEGER
- `token_hash` TEXT
- `issued_at` INTEGER
- `expires_at` INTEGER
- `last_ip` TEXT
- `last_location_id` INTEGER
- `status` TEXT

### 5.5 `auth_logs`
- `id` INTEGER PK
- `user_id` INTEGER NULL
- `ip` TEXT
- `location_id` INTEGER NULL
- `site_id` TEXT NULL
- `decision` TEXT
- `challenge_type` TEXT
- `result` TEXT
- `detail` TEXT
- `created_at` INTEGER

## 6. API 设计（当前实施）
- `GET /auth/health`
- `GET /auth/login`（登录页）
- `GET /auth/location`（地点选择页）
- `GET /auth/check`
- `POST /auth/login`
- `GET /auth/session/context`
- `POST /auth/location/select`
- `GET /auth/locations`
- `POST /auth/locations`
- `GET /auth/bindings`
- `POST /auth/admin/users`（实施阶段便捷接口）

## 7. Nginx 对接要点
- `auth_request` 调用 `GET /auth/check`
- 放行：返回 `204`
- 拒绝：返回 `401` 并返回挑战类型

## 8. 安全设计
- 密码：PBKDF2-HMAC-SHA256 + 随机盐
- TOTP：6 位，30 秒步长，支持 ±1 时间窗口
- Cookie：随机 Token，服务端仅存摘要
- 请求头：IP 优先读取 `X-Forwarded-For`，需在受控代理下使用

## 9. 测试建议
- 单元测试：规则矩阵所有分支。
- 集成测试：登录、换 IP、公共场所策略。
- 安全测试：Cookie 篡改、暴力尝试、伪造 IP 头。
